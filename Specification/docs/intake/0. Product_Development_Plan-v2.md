# Additional Product Decisions Before Development

## Offline / Bad Internet Behavior
The MVP should assume internet is needed for login, syncing, photo upload, and export.  
For version 1, full offline mode is not required.  

**If internet is unavailable, the app should show a clear message:**
* "You are offline. Data cannot be synced right now."
* "Photo upload failed. Try again."

*Do not build complex offline sync in MVP unless specifically requested later.*

---

## App Language
The first version should support English UI by default.  
The app should be designed so Serbian UI can be added later.  

**Voice input should support:**
* Serbian: `sr-RS`
* English: `en-US`

*The user should be able to choose preferred voice language in settings/profile.*

---

## Doctor-Patient Linking
For MVP, doctor-patient linking can be manually created in Supabase.  
Do not build invite codes in the first implementation unless there is time.  

**Later version can add:**
* Doctor creates invite code
* Patient enters invite code
* `doctor_patient_access` row is created automatically

---

## Patient Privacy and Access
Patient owns their own data.  
Doctor can only read data from linked patients.  
Doctor should not edit/delete patient-created entries in MVP.  
Doctor notes should be stored separately from patient entries.

---

## Image Rules
All photos must be standardized before upload.  

**Rules:**
* Do not upload original full-resolution image.
* Resize max width to 1280px.
* Keep aspect ratio.
* Compress as JPEG quality 0.8.
* Generate thumbnail.
* Store image metadata.
* Upload to private Supabase Storage.
* Store only paths/metadata in database.

---

## Voice Input Rules
Voice input is a reusable text input helper.  

**It should:**
* Work across all text fields.
* Append transcript to existing text.
* Let the user edit transcript before saving.
* Never save raw audio.
* Never use paid transcription API in MVP.
* Fall back to manual typing if recognition is unavailable.

*Support Serbian and English recognition where the device/browser allows it.*

---

## Export Rules
The database stores rows.  
Exports generate JSON packages on demand.  

**Support export by:** Selected day, Selected week, Selected month  
**Export modes:** Photos only, Photos + text, Full data  

**JSON should include:**
* patient code
* date range
* entries sorted by timestamp
* entry type
* title
* notes if selected
* custom fields
* photo metadata
* signed photo URLs when needed

*Do not embed base64 images inside JSON.*

---

## Backup / Archive
Because Supabase free storage is limited, add a future plan for archive/export.  
MVP should support monthly export.  
Later, allow old months to be downloaded and optionally deleted from Supabase Storage to free space.

---

## Data Deletion
MVP should include a basic delete-data plan.  

**At minimum:**
* Patient can delete own entries.
* Patient can delete own photos attached to entries.
* Future version should include full account/data deletion.

---

## MVP Definition of Done
Version 1 is considered done when:
1. User can sign up/log in.
2. Patient can create a timestamped entry.
3. Patient can attach a compressed photo.
4. Patient can use voice input where supported.
5. Patient can view daily timeline.
6. Doctor can view linked patient timeline.
7. Doctor can export selected day/month JSON.
8. Supabase RLS protects user data.
9. App runs on Android APK.
10. App runs as a web app.
11. Basic Huawei Android test is performed if a Huawei device is available.

---

## First Development Slice
**The first working slice should be:** `Login -> Patient daily timeline -> Add text entry -> Save to Supabase -> Read from Supabase -> Display on timeline.`

*Only after this works should Codex add photos, voice, doctor dashboard, and exports.*

---

## Testing Plan
**Test with:**
* One patient account
* One doctor account
* One account with bad/empty data
* Android phone
* Huawei phone if available
* Web browser

**Test cases:**
* Create entry
* Edit timestamp
* Delete entry
* Add photo
* Confirm photo compression
* Use voice input
* Voice unavailable fallback
* Doctor views linked patient
* Doctor cannot view unlinked patient
* Export day JSON
* Export month JSON
